# 原文档

一、扩展解决的问题
Scratch 体系的舞台渲染基于精灵合成，制作粒子、辉光、流体一类的连续介质特效时通常只能采用两种手段：以画笔逐帧绘制，或以大量克隆体模拟粒子。前者受制于单线程解释执行的速度，后者受制于克隆体数量上限与碰撞开销，两者都难以在保持帧率的前提下达到现代游戏的视觉水准。

LUMEN Air 将一套完整的 GPU 流体模拟管线（涡度约束、Jacobi 压力求解、半拉格朗日平流、多级 Bloom 辉光）封装为 31 块积木。所有运算在显卡上并行完成，积木层只负责下达指令，因此复杂度与作品自身的脚本量彼此独立：在一个已经很复杂的作品里开启流光层，对原有脚本的执行速度没有可测量的影响。

具体而言，本扩展解决以下几类需求：

视觉表现：发光拖尾、能量爆发、激光束、全屏转场等在原生积木体系内难以实现的效果。
文字演出：标题、得分、对白强调等文字以发光流体的形态出现、保持与消散。
玩法机制：流体不仅可见，而且可读可推——角色能够被流场携带移动，脚本能够查询任意位置的流速与亮度，用于判定与触发。
氛围底噪：作品空闲时的背景动态，单块积木完成，无需常驻循环脚本。
二、适用场景
作品类型 典型用法
节奏 / 音乐 按拍触发全屏闪光与冲击环；判定线用固定文字或光束呈现
跑酷 / 平台 角色持续喷射形成移动拖尾；死亡时冲击环爆发；关卡切换用浪潮转场
弹幕 / 射击 克隆体子弹各自携带喷射拖尾；激光武器以光束积木逐帧绘制；命中点爆发
解谜 / 物理 风场与引力场改变流体走向，角色随流光漂移，亮度传感做区域判定
剧情 / AVG 流动文字的辉光爆发做章节标题；固定文字做常驻字幕；浪潮做场景切换
放置 / 互动挂件 环境流光加鼠标搅动，两块积木构成完整的互动背景
不适用的场景：需要精确到像素的碰撞判定（流体传感的空间分辨率约为 7 个舞台单位）；需要在无 WebGL 环境（极老旧浏览器）运行的作品。

三、快速开始
在扩展库安装 LUMEN Air，或在自定义扩展中粘贴源文件安装。
绿旗脚本中放入「开启流光层 画质[均衡]」。
追加「环境流光[开启] 活跃度(40)」与「鼠标搅动[开启]」。
至此已获得一个可交互的动态流光背景。后续所有积木都建立在流光层开启的前提上；图层未开启时，注入类与参数类积木静默忽略，不产生报错。

四、积木参考
4.1 图层控制
开启流光层 画质[流畅/均衡/高清] 在舞台上方创建流体渲染层。三档画质对应不同的模拟与渲染分辨率：流畅档面向低配置设备与移动端；均衡档为默认推荐；高清档面向全屏展示。重复执行等价于以新画质重建图层，画面清空，但已登记的角色喷射予以保留。

关闭流光层 完整移除渲染层与全部内部状态（含固定文字），释放显存。作品结束或切换到无特效场景时调用。

清空流光 清除流体画面与速度场，图层保持开启。固定文字属于独立图层，此积木对其无效。

平息流场 将速度场与压力场归零，同时撤除风与全部力场。画面（染料）原样保留，一切流动立即静止。典型用途：书写文字前净化环境，防止残留湍流撕散字形；或作为"时间凝固"类演出。

模拟 [继续/暂停] 暂停时流体停止演化但仍然渲染，画面定格；注入类积木在暂停期间仍会改变画面（改动在恢复后开始流动）。与「设置[模拟速度]」配合可实现定格、慢放、快进的完整时间控制。

4.2 注入
所有注入类积木共享同一套力度语义：力度（0–100）同时决定注入速度、光斑尺寸与染料亮度。力度 100 的光斑直径约为力度 0 的四倍，亮度约为五倍。坐标使用 Scratch 坐标系（x −240…240，y −180…180），并允许越出舞台 25%（x 至 ±300，y 至 ±225），用于从画面外发起注入。方向遵循 Scratch 约定：0 为上，90 为右，顺时针增加。

在 x( ) y( ) 朝方向( ) 注入颜色[ ] 力度( ) 基础注入。在指定位置产生一团指定颜色的流光并赋予指定方向的初速度。放入循环并逐帧改变坐标，可以手工构造任意运动轨迹的光源。

在 x( ) y( ) 注入随机霓虹 力度( ) 同上，但颜色从当前霓虹色域主题中随机抽取，方向随机。适合火花、星尘一类不需要精确控制的点缀。

在 x( ) y( ) 引爆冲击环 强度( ) 以指定点为圆心向十四个方向同时注入，形成环形扩散爆发。强度 100 时波及范围约为半个舞台。颜色取自当前色域主题。

从 x1 y1 到 x2 y2 拉一道流光 颜色[ ] 力度( ) 在两点之间生成一条连续光带并沿线注入方向动量。光带由距离场着色器一次绘制完成，无论长短，开销恒定且没有颗粒断点，因此可以放心地每帧调用——持续照射的激光、连接两个角色的能量链都属于这一用法。力度 0 时只画光不推流，可当作发光画笔。

全屏闪光 颜色[ ] 强度( ) 向整个画面均匀注入一层颜色。用于节奏打点、受击反馈、闪电。注入的颜色随后按正常规律消散。

流光浪潮 从[左/右/上/下] 扫过 颜色[ ] 用时( )秒 一道双层结构的光墙（锐利亮芯在前，柔和光晕拖后）从指定边扫过全屏，并推动沿途的既有画面。持续时间 0.2–10 秒。扫屏进行中切换背景或角色布局，即构成完整的场景转场。多道浪潮可以叠加。

4.3 文字
扩展提供两种性质不同的文字，可同屏混用。

用流光写下 [文本] 在 x y 字号( ) 颜色[ ] 字体[ ] ——流动文字 文字以染料形态印入流体，是流体的一部分：会被流动撕扯、随时间消散、参与辉光。同一位置重复执行会使亮度叠加，连续印三次左右即可触发辉光爆发，随后按消散速度衰减。这一特性是刻意保留的：重复次数就是亮度控制手段。 字号单位与 Scratch 坐标一致（字号 60 即字高约 60 个舞台单位，上限 200）。五种字体（黑体、圆体、衬线、楷体、等宽）采用系统字体栈，观众设备缺失对应字体时自动回退到最接近的可用字体。文字渲染自带三层光晕，印入后即具备柔和发光边缘。

写下固定文字 [文本] … (不被搅动) ——固定文字 文字渲染在流体模拟之外的独立图层：与流体共享同一条辉光管线（观感一致），但物理上不在流体之内，任何搅动、风场、爆发都不影响它，也不会随时间消散。 同一坐标重复书写按替换处理——旧文字删除、新文字顶替，因此把它放进循环里逐帧更新变量内容（例如得分）是受支持的用法，不产生叠加发亮或内存增长。静止的固定文字不产生逐帧渲染开销，数十条并存不影响帧率。

擦除所有固定文字 清除全部固定文字。注意管辖范围：流动文字由消散机制或「清空流光」处理，本积木对其无效。

4.4 角色联动
让此角色持续喷射 颜色[ ] 力度( ) 将当前角色登记为持续发射源：角色移动时自动喷出沿运动方向的发光拖尾，静止时缓慢上浮余温。登记一次持续生效，不需要循环维持。克隆体各自独立登记，适合弹幕子弹拖尾。角色被删除时发射器自动注销。执行顺序不敏感：即使此积木先于「开启流光层」运行，登记也会保留到图层开启后生效。

让此角色停止喷射 / 停止所有角色喷射 前者注销当前角色（含克隆体自身），后者注销全部。红色停止按钮等效于执行后者。

让此角色随流光漂移 灵敏度( ) 读取角色所在位置的流场速度并按灵敏度比例移动角色。放入「重复执行」即让角色被流体携带。与风场、引力场组合即得到水流关卡、风力跳跃、黑洞陷阱等机制。灵敏度 50 时，中等强度的流场约以每秒 60 个舞台单位的量级推动角色。

4.5 外力场
设置风 方向( ) 强度( ) / 关闭风 施加全局匀速力场。风持续作用于整个流体，长时间开启会使画面整体流向一侧。

在 x y 放置 [引力场/斥力场] 强度( ) 放置持续作用的点状力场：引力场向内吸聚（漩涡、黑洞），斥力场向外排开（护罩、屏障）。同时最多 8 个，超出时最早放置的被移除。力场本身不可见，通常配合注入积木让流光显形。逐帧「清除所有力场＋重新放置」即可实现移动的力场。

清除所有力场 移除全部点状力场（不影响风）。

4.6 氛围与参数
环境流光 [开启/关闭] 活跃度( ) 开启后引擎按活跃度对应的频率自动在随机位置注入随机霓虹（活跃度 100 约每 0.4 秒一次）。等价于一段自动维护的背景脚本，红色停止按钮会将其关闭。

鼠标搅动 [开启/关闭] 指针划过舞台时沿轨迹注入光带并搅动流体。注意它与作品自身的鼠标交互（点击角色等）共存但不冲突——搅动层不拦截任何鼠标事件。

设置霓虹色域 [霓虹/极光/熔岩/寒冰/彩虹] 限定"随机颜色"（随机注入、冲击环、环境流光、鼠标搅动）的取色范围。彩虹为全色域。

设置 [参数] 为 ( )（0–100） 运行时可调的七个引擎参数：

参数 含义 默认值
消散速度 染料亮度衰减快慢；0 近乎不消散，100 数秒内消失 约 10
漩涡强度 涡度约束力度；高值卷曲绚丽，低值流动平直 34
流动惯性 湍流自持时间；低值下任何扰动数秒内自行平息 约 96
辉光强度 Bloom 增益 50
注入半径 所有注入类积木的基础光斑尺寸 约 26

图层混合模式设为 [发光叠加/不透明覆盖] 发光叠加（默认）下黑色即透明，流光以加亮方式叠在舞台画面上，不遮挡角色；不透明覆盖下流光层连同其黑色背景整体盖住舞台，适合把流光当作独占背景使用。

4.7 传感
传感积木使引擎状态对脚本可见。三个读数块均以低分辨率缓冲回读实现，每帧至多一次显存读取；同一帧内的多次查询共享同一份数据，任意数量的角色同时使用不构成性能负担。空间分辨率约为 7 个舞台单位。

流速x 于 x( ) y( ) / 流速y 于 x( ) y( ) 指定位置的流场速度分量，量纲与注入积木的"力度"一致，便于对照。可用于让脚本感知风向、判断某区域是否处于激流。

流光亮度 于 x( ) y( )（0–100） 指定位置的画面亮度，固定文字的亮度计入在内。典型用法：亮度阈值判定（碰到高亮区域扣血、火焰蔓延到达检测）。

〈流光层已开启?〉 布尔值。推荐所有依赖图层的角色脚本以「等待直到〈流光层已开启?〉」开头，消除绿旗多脚本并发时的时序不确定性。

五、常见组合
互动氛围背景（两块积木的完整方案） 开启流光层 → 环境流光[开启]活跃度(40) → 鼠标搅动[开启]。

章节标题演出 设置[流动惯性]为(25) → 平息流场 → 重复三次「用流光写下[标题]」→ 等待若干秒 → 流光浪潮扫过收尾。低惯性保证标题存续期间环境安定；三连印触发辉光爆发；浪潮同时完成清场与转场。

实时记分牌 角色的重复执行内放「写下固定文字（连接得分：）在固定坐标」。替换语义保证数值刷新而不叠加。

激光武器 发射角色的重复执行内放「从(自身x)(自身y)到(目标x)(目标y)拉一道流光」。光束积木开销恒定，逐帧调用是设计内用法。命中判定另以目标处「流光亮度」阈值实现。

水流关卡 关卡脚本布置风场与力场构成"水路"；玩家角色的重复执行内放「让此角色随流光漂移」；危险区以亮度传感判定。流体既是画面又是关卡机制本身。

爆发定格 引爆冲击环 → 等待 0.5 秒 → 模拟[暂停] → 演出或对白 → 模拟[继续]。爆炸在最大张力的瞬间凝固。

克隆体弹幕拖尾 克隆体启动脚本内放「让此角色持续喷射」，克隆体删除时发射器自动回收，无需清理脚本。

六、技术特性
帧率无关性：消散与衰减按真实时间补偿，60Hz 与 120Hz 设备上观感一致，作品行为不随观众设备刷新率漂移。
连续光带：光束、角色拖尾、鼠标搅动轨迹均由距离场着色器生成，数学上连续，不存在因移动过快产生的断点。
停止语义与 Scratch 一致：红色停止按钮清除全部持续性行为（喷射、搅动、浪潮、环境流光、风、力场），图层与画面保留。
叠加层安全性：渲染层不接收任何鼠标事件，位于舞台画面之上、变量监视器与询问框之下，不干扰作品交互与调试。
兼容性：优先 WebGL2，自动回退 WebGL1。
